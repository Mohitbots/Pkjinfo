## Introduction

CSS has evolved tremendously over the years, introducing powerful features that make styling web pages easier and more efficient. Whether you're a beginner or an experienced developer, these 10 CSS tips will help you write better, cleaner code.

## 1. Use CSS Custom Properties (Variables)

CSS custom properties make your stylesheets more maintainable and themeable.

```css
:root {
  --primary-color: #3b82f6;
  --secondary-color: #f59e0b;
  --spacing-unit: 8px;
  --border-radius: 8px;
}

.button {
  background-color: var(--primary-color);
  padding: calc(var(--spacing-unit) * 2);
  border-radius: var(--border-radius);
}
```

### Why it's useful:
- Easy theme switching
- Better maintainability
- Can be updated via JavaScript
- Reduces code duplication

## 2. Master Flexbox for Layouts

Flexbox makes creating complex layouts simple and intuitive.

```css
.container {
  display: flex;
  justify-content: space-between;
  align-items: center;
  gap: 1rem;
}

.sidebar {
  flex: 0 0 250px; /* Don't grow, don't shrink, 250px wide */
}

.main-content {
  flex: 1; /* Take remaining space */
}
```

### Common Flexbox Patterns:

**Center anything:**
```css
.center {
  display: flex;
  justify-content: center;
  align-items: center;
  min-height: 100vh;
}
```

**Equal-width columns:**
```css
.columns {
  display: flex;
}

.column {
  flex: 1;
}
```

## 3. Use Grid for Complex Layouts

CSS Grid excels at two-dimensional layouts.

```css
.grid-container {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));
  gap: 2rem;
}
```

### Responsive Grid Without Media Queries:
```css
.responsive-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));
  gap: 1.5rem;
}
```

This creates a responsive grid that automatically adjusts the number of columns based on available space!

## 4. Leverage Logical Properties

Logical properties make your CSS work better with different writing modes and languages.

```css
/* Old way */
.element {
  margin-left: 1rem;
  margin-right: 1rem;
  border-left: 1px solid black;
}

/* Modern way */
.element {
  margin-inline: 1rem;
  border-inline-start: 1px solid black;
}
```

### Common Logical Properties:
- `margin-inline` â†’ replaces `margin-left` + `margin-right`
- `padding-block` â†’ replaces `padding-top` + `padding-bottom`
- `inset-inline-start` â†’ replaces `left` (in LTR languages)

## 5. Use Modern Color Functions

Modern CSS color functions provide better control and readability.

```css
.element {
  /* RGB with alpha */
  background-color: rgb(59 130 246 / 0.5);
  
  /* HSL for easier color manipulation */
  color: hsl(220 80% 50%);
  
  /* Relative colors (newer) */
  border-color: hsl(from var(--primary-color) h s calc(l - 20%));
}
```

### Why HSL is Great:
```css
:root {
  --primary-hue: 220;
}

.primary {
  background: hsl(var(--primary-hue) 80% 50%);
}

.primary-light {
  background: hsl(var(--primary-hue) 80% 70%);
}

.primary-dark {
  background: hsl(var(--primary-hue) 80% 30%);
}
```

## 6. Utilize Container Queries

Container queries let components adapt based on their container's size, not the viewport.

```css
.card-container {
  container-type: inline-size;
  container-name: card;
}

.card-title {
  font-size: 1rem;
}

@container card (min-width: 400px) {
  .card-title {
    font-size: 1.5rem;
  }
}
```

This makes truly modular, reusable components!

## 7. Improve Performance with `will-change`

Hint the browser about animations for better performance.

```css
.animated-element {
  will-change: transform, opacity;
}

.animated-element:hover {
  transform: scale(1.1);
  opacity: 0.8;
}
```

## 8. Use `clamp()` for Responsive Typography

`clamp()` creates fluid typography without media queries.

```css
h1 {
  /* Minimum 2rem, preferred 5vw, maximum 4rem */
  font-size: clamp(2rem, 5vw, 4rem);
}

.container {
  /* Responsive padding */
  padding: clamp(1rem, 3vw, 3rem);
}
```

### How it works:
`clamp(MIN, PREFERRED, MAX)`

The browser will use the preferred value unless it's smaller than MIN or larger than MAX.

## 9. Master Modern Selectors

CSS has powerful selectors that reduce the need for extra classes.

```css
/* :is() - reduces repetition */
:is(h1, h2, h3, h4, h5, h6) {
  font-weight: bold;
  line-height: 1.2;
}

/* :where() - zero specificity */
:where(article, section) > p {
  margin-block: 1em;
}

/* :has() - parent selector! */
.card:has(img) {
  display: grid;
  grid-template-columns: 200px 1fr;
}

/* :not() - exclude elements */
li:not(:last-child) {
  border-bottom: 1px solid gray;
}
```

### The `:has()` selector is game-changing:
```css
/* Style form if it has invalid inputs */
form:has(input:invalid) {
  border: 2px solid red;
}

/* Style parent based on child state */
.product:has(.badge.sale) {
  background-color: yellow;
}
```

## 10. Optimize with `aspect-ratio`

Maintain consistent aspect ratios without padding hacks.

```css
.video-container {
  aspect-ratio: 16 / 9;
  width: 100%;
}

.square-image {
  aspect-ratio: 1;
  object-fit: cover;
}

.profile-pic {
  aspect-ratio: 1;
  width: 100px;
  border-radius: 50%;
}
```

### Before `aspect-ratio`:
```css
/* The old padding hack */
.video-wrapper {
  position: relative;
  padding-bottom: 56.25%; /* 16:9 ratio */
  height: 0;
}

.video {
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
}
```

Much cleaner now!

## Bonus Tips

### Use `object-fit` for Images
```css
img {
  width: 100%;
  height: 300px;
  object-fit: cover; /* or contain, fill, none, scale-down */
  object-position: center top;
}
```

### Smooth Scrolling
```css
html {
  scroll-behavior: smooth;
}
```

### Better Focus Indicators
```css
:focus-visible {
  outline: 2px solid var(--primary-color);
  outline-offset: 2px;
}
```

### Prevent Layout Shift with `content-visibility`
```css
.off-screen-content {
  content-visibility: auto;
}
```

## Practical Example: Putting It All Together

Here's a modern card component using these techniques:

```css
.card {
  /* Container query */
  container-type: inline-size;
  
  /* Layout */
  display: grid;
  gap: 1rem;
  
  /* Spacing with logical properties */
  padding-block: 1.5rem;
  padding-inline: 1.5rem;
  
  /* Colors with custom properties */
  background-color: var(--card-bg);
  border: 1px solid var(--card-border);
  border-radius: var(--border-radius);
  
  /* Smooth transitions */
  transition: transform 0.2s ease;
}

.card:hover {
  transform: translateY(-4px);
}

.card-image {
  aspect-ratio: 16 / 9;
  object-fit: cover;
  border-radius: var(--border-radius);
}

.card-title {
  font-size: clamp(1.25rem, 3vw, 1.75rem);
  margin-block-end: 0.5rem;
}

@container (min-width: 600px) {
  .card {
    grid-template-columns: 200px 1fr;
  }
}
```
